## Design Guidelines

### Visual Consistency
- **Color scheme:** Professional blues/greens for trust
- **Confidence colors:** Green (high), Yellow (medium), Red (low)
- **Font:** Clean, modern sans-serif
- **Logo placement:** Consistent across all assets

### Data to Use in Demos
- **Always use realistic numbers:** Based on actual use cases
- **Highlight key metrics:** 10x faster, 90% cheaper, 70% more accurate
- **Show real companies:** Use "[Industry Leader]" if can't name
- **Include timestamps:** Prove real-time processing

### Screenshot Best Practices
- **High resolution:** Minimum 1920x1080
- **Clean data:** No test entries or Lorem ipsum
- **Highlight key features:** Use arrows or boxes sparingly
- **Remove sensitive info:** Blur customer names if needed

### For Pharma Demo:

- Include recent approvals for timeliness

### For Conference Demo:

### For Citation Checking:
- Use excerpt from published paper

## Presentation Tips

1. **Always start with the problem:** Show manual chaos first
2. **Demo live when possible:** Builds trust and credibility
3. **Have backup screenshots:** In case of technical issues
4. **Keep demos short:** 2-3 minutes maximum
5. **Focus on outcomes:** Time saved, money saved, accuracy gained
